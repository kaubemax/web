3 · How an X-grid becomes a rose

Medium. The image imitates counted cross-stitch embroidery. Every "pixel" is a small X — two crossing diagonal stitches — placed on a fixed square grid (10 px spacing, each X about 8 px). Nothing is anti-aliased or blended; colour only changes from one whole stitch to the next, so the rose is built entirely out of discrete coloured crosses on a transparent / cream background.

Geometry of one rose. A rose occupies an ellipse. For each grid cell inside it we take polar coordinates — radius r (0 at the centre, 1 at the rim) and angle θ — and decide the stitch colour like this:

  • Silhouette. The outline is a soft 5-lobe scallop (edge = 0.86 + 0.14·cos(5θ)), so the flower reads as five rounded petals rather than a circle.
  • Petal rings. The radius is split into 3 concentric layers. Each layer is rotated by half a petal relative to the one inside it, so petals interleave like a real rose instead of lining up into spokes.
  • Shading. Within each ring the colour runs dark at the petal base → light at the petal tip, and the angular creases between petals are darkened further. Overall the bloom goes from a deep, shadowed heart to bright, light-catching outer petals.
  • Heart. The innermost core is a tight dark spiral swirl — the rolled centre of the bud.

The palette is a 5-step ramp from darkest to lightest, chosen per rose so some are deep red and some are blush-pink. Leaves and the connecting vine use greens.

5 · What we want improved

Keep the counted cross-stitch style — discrete coloured X stitches on a square grid, cream/transparent background, the warm rose-and-green palette above. But make the roses read more convincingly as roses: clearer layered, overlapping petals; a believable spiral bud at the centre; natural highlights and shadows so the bloom looks three-dimensional; clean petal edges rather than a mottled blob. The result should still be expressible as a grid of coloured crosses (so it can be re-encoded back into this stitch renderer).
